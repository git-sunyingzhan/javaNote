1、接口
    实际上就是一个规范。他会要求你去做什么，但是不会要求你具体怎么做。
    接口里面定义的是额外功能，但是不给出具体实现。
    格式：
        interface 接口名{}
        interface 接口名 extends 接口名{}
        interface 接口名 extends 接口名，接口名{}
    实现接口
        class 类名 implements 接口名{}
        class 类名 implements 接口名，接口名{}
    接口可以实例化吗？
        接口不可以实例化
    接口可以多继承吗？
        可以
    一个类可以实现多个接口吗？
        可以
    接口的实现类
        抽象类：根据实际需求判断到底是否需要重写接口内的抽象方法
        具体类：必须要重写接口里面所有的抽象方法（如果接口里面的抽象方法，有重名，只需要重写一次）
    接口的组成
        1）成员变量
            只能是常量，默认修饰符public static final(可以省略)
        2）构造方法
            没有
        3）成员方法
            只能是抽象方法，默认修饰符public abstract
扩展：
    jdk1.8可以在接口中可以使用默认方法和静态方法。
    jdk1.9接口中支持私有方法。
    默认方法：供实现类调用，或实现类重写
        1）接口中有多个默认方法时，实现类都可以直接使用。如果默认方法有重名，实现类需要重写
        2）当一个类，即继承了一个父类，又实现了多个接口，如果父类中的成员方法与接口中的默认方法重名，
            子类就近选择父类里面的方法。
    静态方法：供接口直接调用
    私有方法：
        普通私有方法：供默认方法调用
        静态私有方法：供静态方法调用
        如果一个接口中，有多个默认方法，并且方法中存在冗余代码。这时候需要将冗余代码封装到一个单独的私有方法中，
        供默认方法去调用。从设计角度讲，私有方法是对默认方法和静态方法的辅助。
2、类和类、类和接口、接口和接口之间的关系
    类和类
        继承关系，只能单继承、多级继承，不可以多继承
    类和接口
        实现关系，可以单实现、多实现
    接口和接口
        继承关系，可以单继承、多继承、多级继承
3、抽象类和接口的区别
    接口是一种特殊的抽象类
    1）成员组成不同
        抽象类：成员变量（变量、常量）、构造方法、成员方法（具体、抽象）
        接口：常量和抽象方法，jdk1.8默认方法和静态方法，jdk1.9私有方法
    2）关系不同
        抽象类与其他类，继承关系
        接口和其他类，实现关系
    3）设计理念不同
        抽象类：多个不同种类事物的统称，共性功能
        接口：放入额外功能，扩展功能

4、权限修饰符
            public protected  default   private
    当前类    可以      可以       可以      可以
    同一类    可以      可以       可以      不可以
    子父类    可以      可以       不可以    不可以
    不同类    可以      不可以      不可以    不可以
